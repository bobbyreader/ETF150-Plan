```
# Project: 长赢 150 计划投资决策辅助工具开发需求

## 1. 项目背景与目标
开发一套基于“ETF拯救世界”（e大）长赢指数投资体系的量化辅助工具。该工具旨在通过实时与历史数据分析，判断全市场及特定指数的估值水位，并根据 150 计划的“存量资金管理”逻辑生成买入/卖出信号及仓位建议。

## 2. 核心估值算法逻辑 (Valuation Engine)
系统需实现以下自定义估值计算模型，严禁直接套用交易所公布的市值加权数据：

- **等权 PE (Equal Weight PE TTM)：**
  - 计算公式：针对特定指数（如沪深300、中证500）的所有成分股，计算其 PE(TTM) 的简单算术平均值或中位数。
  - **剔除机制：** 计算前必须剔除 PE 为负（亏损股）以及 PE > 150（极端泡沫股）的样本。
- **全市场 PB 锚点：**
  - 监控全市场 PB(市净率)。PB < 2.0 标记为低估区域，PB < 1.6 标记为历史超级大底/极限区域。
- **历史百分位计算：**
  - 获取过去 5 年和 10 年的每日 PE/PB 序列。
  - 当前值在历史序列中的位置即为“百分位”。
  - **价值区域划分：**
    - 钻石区域：百分位 < 10%。
    - 黄金区域：百分位 < 20%。
    - 正常区域：百分位 20% - 80%。
    - 风险区域（哈迪斯之顶）：百分位 > 80% 或 全市场 PE > 46 倍。

## 3. 仓位控制算法 (Position Sizing)
系统需根据全市场百分位自动计算“目标权益仓位”：

- **基准公式：** 目标权益仓位 = 100% - 当前百分位。
- **动态调节逻辑：**
  - **高估调节（百分位 > 50%）：** 目标权益仓位 = (100% - 当前百分位) / 2。
  - **中枢回归：** 当百分位降至 50% 附近，迅速补足权益仓位至 40%-45%。
  - **低估放大（百分位 < 35%）：** 在基准公式基础上“艺术性放大”投入（例如百分位 30% 时，配比可从 70% 提升至 75%-80%）。

## 4. 150 计划份数分配模块 (Unit Logic)
- **资金定义：** 将总存量资金均分为 150 份。
- **发车逻辑：**
  - 常规买入：每次买入 1 份。
  - 加速买入（黄金/钻石底）：单次买入 2-4 份。
  - 极端底部：在全市场 PB 接近 1.6 的区域，单月最高可投入 40 份。
- **内部换马 (Horse Swapping)：** 监控同大类品种（如中证500与创业板）的估值差，当一者高估而另一者低估时，提示卖出高估品种并买入低估品种。

## 5. 风险预估与交易细节模块
- **预期最大跌幅 (Max Drawdown Projection)：** 每个信号必须输出“预计最大跌幅”。
  - 证券/金融类：预设 50%-60%。
  - 创业板：预设 40%-60%。
  - 中证500/红利：预设 25%-40%。
- **IOPV 监控：** 实时抓取场内 ETF 的实时参考净值 (IOPV)，若现价高于 IOPV 则提示“暂不买入”，避免支付溢价。

## 6. 功能模块要求
1. **多市场面板：** 同时显示 A 股、港股、海外市场（美、德）及商品（黄金、原油）的估值百分位。
2. **资产配置比例图：** 生成包含权益（大盘/中小盘/行业）、固收（转债/纯债）、海外、商品的扇形图。
3. **S 定投模式切换：** 提供针对增量资金的模式，每月月初固定提示买入 1-2 份。
4. **历史回测：** 支持查询特定指数在不同估值起点买入的 3/5/10 年年化收益率对比。

## 7. 技术栈建议 (可选)
- **后端：** Python (Pandas, NumPy)
- **数据源：** 优先接入 AkShare, Tushare 或类似金融数据 API
- **前端演示：** Streamlit 或 Flask
```

### 给开发者的执行建议（基于源码精神）：

- **不预测，只计算：** 工具不应包含涨跌预测功能，仅反映当前价格相对于历史价值的位置。
- **压力测试为王：** 必须在界面醒目位置显示“最坏情况发生后账户会怎样”，以管理用户心理预期。
- **剔除权重干扰：** 在计算全市场 PE 时，务必通过等权法或中位数法，以此打破大市值银行股对真实估值水位的“伪装”。
